# HCI 后端核心日志执行计划

版本：v0.4  
日期：2026-06-10  
目标：面向稳定期精神分裂症/精神障碍人群的 HCI 与临床融合研究，只记录高效、有效、可解释、低隐私风险的数据。数据应能真实说明患者用了什么模式、留下了多少作品、作品经历了多少次修改/生成，以及不同模式产生了哪些可分析的环境表达指标。

## 1. 最终记录原则

当前研究后端数据遵循以下原则：

1. 不记录鼠标轨迹、hover 热区、连续拖拽坐标、逐秒行为流。
2. 不开发程序内量表系统，不保存逐题量表答案。
3. 不记录研究员观察日志。
4. 不记录安全事件日志；安全事件如有需要，由外部伦理/临床研究表单记录。
5. 暂不建立“作品与环境输出汇总表”。
6. 只记录能在论文中有理有据解释的核心摘要数据。
7. 面向研究人员的数据看板、飞书表名、飞书字段名和导出说明优先使用中文显示；后端字段、数据库列名和同步类型可保留英文技术名，但必须在文档中配中文解释。
8. 如果遇到飞书自动生成的英文或乱码字段，应尽量改成中文提示名；无法删除但无需分析的系统字段统一标注为 `系统主列_忽略`。

## 3. 已加入的核心字段

### 3.1 草稿箱作品数量与修改/生成次数

这是本轮确定的新增核心记录口径，用来替代之前散乱的“主动继续次数”“保存作品次数”“完成状态”等字段。

本地表：`hci_work_count_summaries`。  
飞书表显示名：`作品数量汇总`。  
飞书表 ID：`tblsa0bsFu7I1tHs`。  
每名参与者一行，按四个模式分别统计。

| 字段名 | 中文解释 | 记录规则 |
| --- | --- | --- |
| `summary_key` | 汇总键 | 飞书唯一键，当前等于本地用户 ID，用于稳定 upsert |
| `participant_code` | 参与者编号 | 匿名研究编号，用于和外部量表合并 |
| `user_id` | 本地用户 ID | 本地排查用，不作为论文核心变量 |
| `display_name` | 登记姓名 | 研究人员登记的姓名或代称，用于现场核对 |
| `total_draft_work_count` | 草稿箱作品总数 | 该参与者草稿箱中全部作品/项目数量 |
| `drag_work_count` | 自由创作作品数 | 草稿箱中来自自由创作模式的作品数量 |
| `drag_revision_generation_count` | 自由创作修改/生成次数 | 自由创作模式下，保存草稿或发起 AI 生成均计 1 次；不记录拖动过程 |
| `inspire_work_count` | 灵感创想作品数 | 草稿箱中来自灵感创想模式的作品数量 |
| `inspire_revision_generation_count` | 灵感创想修改/生成次数 | 灵感创想模式下，保存草图草稿或发起 AI 生成均计 1 次 |
| `chat_work_count` | 对话改造作品数 | 草稿箱中来自对话改造模式的作品数量 |
| `chat_revision_generation_count` | 对话改造修改/生成次数 | 对话改造模式下，每次保存结果或发起 AI 生成计 1 次 |
| `slider_work_count` | 智能参数作品数 | 草稿箱中来自智能参数模式的作品数量 |
| `slider_revision_generation_count` | 智能参数修改/生成次数 | 智能参数模式下，每次保存结果或发起 AI 生成计 1 次 |
| `updated_at` | 更新时间 | 统计刷新时间 |

解释口径：

- “作品数”只看草稿箱中实际存在的作品/项目数量。
- “修改/生成次数”记录作品层面的有效迭代，不记录每次拖动、每次输入、每次滑杆微调。
- 一次 AI 生成算一次修改/生成。
- 保存草稿也算一次修改/生成，因为它代表用户完成了一次有意义的作品状态更新。
- 这两个指标分开记录：一个参与者可以只有 1 个作品，但该作品经历 5 次修改/生成。

## 4. 四个环境参数的判定口径

四个环境参数是程序用于控制 AI 环境改造方向的设计参数，不是临床症状量表：

- 绿化程度：希望图像中增加或保留多少树木、草地、灌木、花卉等自然植被。
- 城市化程度：希望图像中出现多少硬质铺装、建筑、长椅、路灯、围栏、道路等人工设施。
- 活力程度：希望环境呈现安静/少人，还是更有活动、人群、社交与动态感。
- 光照温暖度：希望画面偏冷、自然日光，还是偏暖、柔和、治愈性的光照。

在智能参数模式中，这四个值会被后端转换为发送给 AI 图像生成服务的提示词片段。例如高绿化会对应“大幅增加树木、灌木、藤蔓和草坪”，高活力会对应“有行人散步、交谈，整体热闹有活力”。因此它们既是系统操控变量，也是最终提示词的结构化来源。

| 模式 | 绿化、城市化、活力、光照温暖度怎么判定 |
| --- | --- |
| 智能参数模式 | 直接记录用户滑杆数值或后端实际生成参数 |
| 自由创作模式 | 暂不记录四个环境参数，只记录植物类数量、其他元素数量、总元素数量、作品数量、修改/生成次数 |
| 灵感创想模式 | 有明确系统识别或参数推断时可记录；没有明确来源时不强行打分 |
| 对话改造模式 | 只保存 AI 最终用于生成图像的四个环境参数；如果没有稳定参数转换，则暂不记录 |

论文表述建议：

> 对于智能参数模式，系统直接记录绿化、城市化、活力与光照温暖度四个环境参数；对于自由创作模式，系统主要记录可观察的元素构成指标；对于灵感创想与对话改造模式，系统仅在存在明确系统识别或最终生成参数时记录环境参数，避免对患者表达作过度推断。

## 5. 明确不加入的字段/表

根据 2026-06-09 口径，以下内容不加入程序后端：

| 内容 | 不加入原因 |
| --- | --- |
| 作品与环境输出汇总表 | 暂时不需要，避免重复存储 |
| `staff_observation_logs` | 研究员观察如需记录，放在外部 CRF/Excel/纸质表 |
| `safety_events` | 安全事件遵循伦理和临床研究流程，外部记录 |
| 程序内量表系统 | WHO-5、PANSS-6、CDSS、PSP 等外部单独收集 |
| 主动继续次数 | 定义不稳定，容易主观化 |
| 研究员帮助等级 | 暂不记录，避免增加现场负担 |
| 每次拖动、hover、撤回、坐标轨迹 | 隐私风险和噪声高，论文解释价值低 |
| `session_summary` 文本摘要 | 暂不需要，统计分析价值有限 |

## 7. Study 2/Study 3 中的使用方式

Study 2 四模式对比时，核心后端变量建议为：

- 草稿箱作品总数。
- 四个模式分别的作品数。
- 四个模式分别的修改/生成次数。
- 自由创作的植物类数量、其他元素数量、总元素数量。
- 灵感创想的系统识别植物/其他数量、用户自定义类别数量。
- 智能参数模式的四个环境参数。
- 对话改造模式中 AI 最终使用的四个环境参数。

Study 3 社区短期部署时，核心后端变量建议为：

- 患者是否持续产生作品。
- 哪个模式产生作品最多。
- 哪个模式经历更多修改/生成。
- 自由创作和灵感创想中植物类表达是否更常见。
- 智能参数/对话模式中最终环境参数是否呈现偏自然、偏安静、偏暖光等趋势。
