# 论文脉络与顶刊评估：CHI 稳定期精神分裂症专版 AIREC

## 1. 论文定位：一句话说清楚

AI 辅助恢复性环境共创活动如何在稳定期精神分裂症康复场景中，通过低负担表达、可见反馈、作品完成和工作人员陪伴，支持患者的表达与控制感、恢复性心理舒适感、感知康复支持和康复沟通。

本文不证明单次实验可以改善精神分裂症症状。它可以测量患者是否在活动后主观上更平静、更舒服、更有希望、更愿意做康复相关的小活动。这些是感知心理支持和即时主观心理状态，是 HCI 论文中合理且有价值的体验结局。

游戏化不作为论文中心目的，而是作为解释参与的机制。论文中心应从“设计了一个好玩的共创游戏”转向“设计了一种能让患者更容易表达并获得心理支持感的恢复性环境共创活动”。

## 2. 论文主线和机制链条

| 机制环节 | 在 AIREC 中的体现 | 心理健康或康复意义 |
| --- | --- | --- |
| 低负担入口 | 拖拽、涂画、对话、参数四种入口，研究员可陪伴读题和辅助。 | 降低数字操作和语言表达造成的挫败。 |
| 可见反馈 | AI 把选择和描述转化为环境图像。 | 让患者看到自己的表达产生结果，增强被理解感。 |
| 控制与选择 | 患者可以选择、修改、保存和解释作品。 | 支持自主感和控制感。 |
| 恢复性环境想象 | 图像围绕平静、安全、活力、陪伴等情绪目标展开。 | 形成即时平静感、舒适感和安全感。 |
| 作品完成 | 最终环境图像作为小作品保存。 | 形成完成感和胜任感。 |
| 康复沟通 | 作品可用于与工作人员讨论偏好、需要和活动计划。 | 把一次数字活动连接到真实康复服务。 |

建议机制链条：低负担共创任务 → 可见反馈和环境作品 → 控制感与被理解感 → 恢复性心理舒适感和主观康复支持感 → 康复沟通与活动计划。

## 3. 数据来源与结局一一对应

| 阶段 | 数据 | 服务的论文结局 |
| --- | --- | --- |
| 前测 | WHO-5、一般数字信心、即时心理状态 4 项；医生评估 PANSS-6、CDSS、PSP。 | 描述样本；控制基线症状和功能差异；提供实验前心理状态。 |
| 四模式体验 | 每个模式 8-10 分钟，拉丁方平衡。 | 比较不同交互入口的体验差异。 |
| 每模式后 | 模式后 11 项。 | 比较负担、表达与控制、个人相关性、恢复性心理舒适感、继续尝试和安全不适。 |
| 结束后 | 即时心理状态 4 项、UMUX-LITE、AIM 友好版、感知心理支持与康复沟通 6 项、偏好排序和访谈。 | 评价总体心理支持、可接受性和康复沟通价值。 |
| 工作人员 | IAM、FIM 和结构化访谈。 | 评价实际康复场景中的适配性、可行性和风险。 |
| 后台数据 | 生成次数、主动继续、作品保存、辅助程度、疲劳程度、AI 失败、图像接近度和安全事件。后台不记录量表答案。 | 解释行为层面的探索、坚持、完成与风险。 |

## 5. 研究问题与假设

| 编号 | 研究问题 | 对应数据与分析 |
| --- | --- | --- |
| RQ1 | 稳定期精神分裂症患者能否在研究员陪伴下完成四种 AIREC 交互模式，且不产生明显不可承受的不适？ | 完成率、辅助程度、安全事件；描述统计和逻辑模型。 |
| RQ2 | 四种模式在表达与控制感、负担和恢复性心理舒适感上有何差异？ | 模式后主结局与次要结局；LMM 主分析。 |
| RQ3 | 单次 AIREC 体验前后，患者的即时平静、舒适、希望和康复参与意愿是否发生有利变化？ | 实验前后 S1-S4；配对 t 检验或 Wilcoxon；探索性分析。 |
| RQ4 | 最终环境作品是否有助于患者表达偏好，并促进与工作人员的康复沟通？ | 患者访谈、工作人员访谈、作品回顾；主题分析和 joint display。 |
| RQ5 | 会话级后台数据能否在不记录轨迹级行为的前提下解释参与、完成和安全差异？ | 生成次数、主动继续、作品保存、AI 错误、安全事件；相关或负二项模型。 |

## 6. 论文逐段写作方向

| 段落 | 写作内容方向 |
| --- | --- |
| Abstract | 第一句写稳定期精神分裂症康复中存在表达困难、动机不足和低参与问题；第二句写 AIREC 是低负担的 AI 恢复性环境共创活动；第三句写单次混合方法和四模式组内研究；第四句写核心发现，包括表达与控制、恢复性心理舒适感和康复沟通；第五句写贡献。 |
| Introduction P1 | 从稳定期精神分裂症治疗后的未满足需求切入：症状稳定不等于康复完成，仍有阴性症状、社会退缩、动机不足、表达困难和功能恢复需求。 |
| Introduction P2 | 说明数字心理健康工具和环境疗愈有潜力，但许多系统仍是被动观看或文本输入，难以支持表达困难人群。 |
| Introduction P3 | 引入 AI 共创：它能把拖拽、涂画、对话和参数调节转化为可见环境作品，但必须有人工监督、低刺激设计和数据最小化。 |
| Introduction P4 | 引入心理健康感知结局：单次 HCI 实验可以评估当下平静感、舒适感、希望感、康复支持感和沟通意愿，而不是宣称临床疗效。 |
| Introduction P5 | 提出研究问题：四种模式如何影响负担、表达与控制、恢复性心理舒适感、参与机制和康复沟通。 |
| Related Work | 分为数字精神康复与 HCI、恢复性环境与数字自然、AI 艺术共创、严肃游戏作为参与机制、严重精神障碍可及性、脆弱人群数据伦理。 |
| System/Design Goals | 写清四种模式如何对应不同表达入口：拖拽是操作探索，涂画是视觉想象，对话是语言叙事，参数是策略调节。强调低刺激、无排行榜、无竞争奖励。 |
| Study Methods | 写清样本、稳定期定义、拉丁方、每模式时长、谁填量表、谁做评估、谁观察、量表外部采集、后台只记录会话摘要。 |
| Findings | 建议顺序：可行性与完成率；主结局模式差异；即时状态探索性变化；后台行为；作品和访谈；工作人员视角；安全事件。 |
| Discussion | 提出 recovery-oriented co-creation 框架，解释 AI 共创活动如何在不宣称临床疗效的前提下支持表达、控制、平静感、希望感和康复沟通。 |

## 7. 结局分析方法：逐项解释

| 结局/数据 | 分析方法 | 为什么用 | 报告方式 |
| --- | --- | --- | --- |
| 表达与控制 | LMM，线性混合效应模型。 | 同一患者体验四种模式，评分存在重复测量，LMM 可控制个体差异。 | 评分 = 模式 + 顺序 + 任务 + 数字信心 + (1\|患者)。报告均值差、95%CI、效应量。 |
| 恢复性心理舒适感 | LMM。 | 比较四种模式对恢复性心理舒适感的差异。 | 报告哪种模式更能产生心理舒适和感知支持，但不写成疗效。 |
| 负担、继续尝试 | LMM 或有序模型。 | 比较模式在负担和继续尝试意愿上的差异。 | 报告均值差或有序模型估计值。 |
| 实验前后即时心理状态 | 配对 t 检验或 Wilcoxon 符号秩检验。 | 同一患者实验前后 S1-S4 的变化是配对数据。 | 报告均值差、95%CI、p 值、效应量，并强调是探索性即时变化。 |
| 参与机制与后台行为 | 负二项模型或 Spearman 相关。 | 生成次数、主动继续和作品保存是计数变量，常偏态。 | 解释探索、坚持和完成是否与心理舒适感相关。 |
| 完成率和安全事件 | 描述统计、逻辑回归或混合逻辑模型。 | 完成/不适是二分类或事件数据。 | 报告事件率、严重度、是否与 AI 输出有关和处理结果。 |
| 访谈和作品 | 主题分析或框架分析。 | 用于解释量化差异背后的机制。 | 用 joint display 对应量化结果、代表性访谈、作品特征和设计启示。 |
| MMRM | 重复测量混合模型，主要用于 RCT 多时间点疗效分析。 | 本研究是单次 CHI 实验，不作为主分析。 | 文件中出现 MMRM 时，应明确为 RCT 术语。 |

## 8. 量表与题项严谨性处理

| 问题 | 修订方案 | 论文解释 |
| --- | --- | --- |
| 成熟量表 | WHO-5、PANSS-6、CDSS、PSP、UMUX-LITE、IAM、FIM 保留来源和评分规则。 | 用于基线描述、可用性、工作人员实施评价。 |
| 患者友好改写 | UMUX-LITE 和 AIM 患者版使用语义等价或构念一致的中文友好表达。 | 论文中说明为中文患者友好化表达。 |
| 自拟题项 | 数字信心、模式后题项、感知心理支持与康复沟通 6 项均表述为 study-specific items。 | 用于测量本系统相关的近端体验，不称为已验证临床量表。 |
| 内容效度 | 正式实验前由精神科/康复/HCI 专家审阅题项。 | 记录专家反馈和修改过程；必要时计算 I-CVI。 |
| 认知访谈 | 3-8 名稳定期患者试读或听读题项，检查理解度、歧义和不适。 | 证明患者能理解题目。 |
| 内部一致性 | 合成维度时报告 Cronbach’s α 或 McDonald’s ω；2 项维度报告两题相关或 Spearman-Brown。 | 若一致性不足，改为单项报告。 |

## 9. 可能结果与顶刊判断

| 结果类型 | 可能发现 | 写作策略 |
| --- | --- | --- |
| 强结果 | 四模式差异清晰；部分模式在表达与控制、恢复性心理舒适感上更好；安全事件少；访谈支持康复沟通价值。 | 具备较强 CHI 潜力。贡献可从工具可用性上升为恢复性环境共创如何支持心理健康感知。 |
| 心理健康感知阳性但模式差异弱 | 患者整体觉得更平静、更舒服、更愿意沟通，但四模式差异不强。 | 可写成系统整体机制论文，强调作品、访谈和工作人员视角。 |
| 工具阳性但心理健康弱 | 控制感和表达支持较好，但平静、希望和康复支持感不突出。 | 回到 AI 环境共创与表达支持主线，不夸大心理健康意义。 |
| 游戏机制阳性但心理健康弱 | 兴趣、完成感较好，但心理健康感知不明显。 | 降低游戏化权重，避免把论文写成单纯游戏体验研究。 |
| 安全问题 | 部分患者出现紧张、困惑、被监控感或 AI 输出不适。 | 写为设计边界，降低刺激、缩短任务、加强工作人员辅助。 |

## 11. 潜在问题

| 潜在问题 | 回答方向 |
| --- | --- |
| 为什么不是 RCT 还要问心理健康？ | 本研究测量的是即时主观心理状态和感知康复支持，不是临床疗效。HCI 研究可以评估系统是否让用户当下更平静、更舒服、更有希望。 |
| 是否夸大康复效果？ | 不使用“治愈”“症状改善”等表述。使用 perceived psychological support、perceived recovery support、immediate affective state 等表述。 |
| 为什么还保留游戏化？ | 游戏化只作为参与机制，解释兴趣、完成感和持续尝试，不作为论文最终价值。最终价值是心理支持和康复沟通。 |
| 量表是否过多？ | 患者每模式后 11 项，结束后量表控制在短版；临床量表由评估者完成，不让患者自填。 |
| 为什么量表不放在程序内？ | 减少操作负担，避免游戏任务和研究问卷混杂，也便于 REDCap、问卷星、飞书表单或纸笔 CRF 规范化管理。 |
| LMM 和 MMRM 是什么？ | LMM 用于同一患者四个模式后的评分比较；MMRM 主要用于 RCT 多时间点疗效分析，本单次 CHI 研究不用 MMRM 作为主模型。 |
| 自拟题项信效度如何解释？ | 表述为 study-specific items，用于测量本系统相关的近端主观体验；研究前由精神科/康复/HCI 专家审阅，并通过患者认知访谈检查理解度；合并维度时报告 Cronbach’s α 或 McDonald’s ω；不把自拟题项称为已验证临床量表。 |
