# 灵感创想功能审查与实验化改造记录

## 代码与功能审查

- 原页面曾依赖 `app/pages/inspire_mode.py` 中的 `.pyc` 恢复壳，难以审查、调试和长期维护；已替换为可维护源码。
- 当前链路被明确为：上传原图 -> 手绘笔迹 -> 自动形状/区域/颜色识别 -> 用户标注修正 -> 场景意图组合 -> prompt 生成 -> 生成图 -> 行为与草图数据导出。
- 已保留原有 `SketchAnalyzer` 与 `SketchComposer`，新页面只负责交互编排和研究数据结构化。

## Bug 与结构优化

- 将灵感创想画布逻辑拆到 `app/static/inspire_canvas.js`，减少 Python 页面中的内联脚本体积。
- 生成按钮在空画布时会被拦截，避免无笔迹生成。
- Fabric.js 已固定为本地静态资源，并保留画布引擎加载失败提示。
- 保存和生成都会持久化草图 JSON、画布快照、自动识别结果、用户标注和 prompt。

## 研究数据补强

- `sketch_data` 中新增 `strokeLog`、`userAnnotations`、`sceneIntent`、`hciMetrics`。
- 导出层应包含自动识别数量、用户修正数量、创意方向、AI 采纳强度、交互轮次、画布路径与快照路径。
- 匿名化与同意机制本轮暂不实施，按当前需求保留后续处理。

## 面向 HCI 顶会的增强方向

- 可解释 AI：每笔会显示形状、区域、颜色、置信度与用户标注覆盖关系。
- 用户能动性：用户可在选择模式中修正 AI 对笔画的理解。
- 多样性保护：提供“疗愈自然 / 自由野趣 / 克制写实”三种创意方向，并写入生成 prompt 和研究数据。
- 可实验变量：`creativeLens` 与 `aiAgency` 可作为实验条件或协变量，用于研究 AI 主导程度、创意多样性和用户控制感。
