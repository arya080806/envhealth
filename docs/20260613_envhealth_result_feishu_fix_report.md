# 2026-06-13 envhealth.cn 结果页与飞书同步修复报告

## 一、问题范围

本轮针对 `envhealth.cn` 当前入口反馈的问题做第一轮底层修复，重点覆盖：

1. 结果详情页生成图加载慢、显示不清晰。
2. “改造前 vs 改造后”模块需要改为大图展示改造后，并用滑杆查看原图。
3. 草稿箱所有模式的结果详情页需要统一应用上述交互。
4. “元素布局 / 创作画布”图片不够清晰，且历史来源可能混入最终生成图。
5. 四种模式的用户操作记录需要更稳地进入对应飞书子表。

## 二、已完成修复

### 1. 图片加载与清晰度

- 新增 `app/services/image_variants.py`，统一管理缩略图和展示图生成。
- 展示图从 `1280x1280 / quality=82` 提升为 `1920x1920 / quality=88`。
- 上传图、AI 生成图、画布快照保存后会立即预热 `thumb/display` 缓存，避免患者第一次打开详情页时才临时压缩图片。
- `/api/image/{filename}?display=1` 和 `?thumb=1` 改为复用统一图片变体服务。

### 2. 结果详情页交互

- `app/pages/result.py` 的共用结果页已改造，草稿箱内所有模式都会使用同一套展示。
- 原来的左右两列“改造前 vs 改造后”改为：
  - 默认显示改造后的大图。
  - 滑动条向右滑动可查看原图覆盖区域。
  - 右上角新增“查看原图”按钮，点击打开原图大图。
  - 长按改造后大图区域会触发下载改造后图片。
- 结果大图和创作画布图都使用更高清的展示图，并提升了画布展示尺寸。

### 3. 元素布局历史修正

- “元素布局 / 创作画布”模块现在只接受文件名中包含 `_canvas_` 的真实画布快照。
- 避免从生成历史中误把 `_out_` 最终结果图混入画布历史，降低“元素摆放图显示成最后生成图”的风险。

### 4. 飞书同步字段补强

飞书仍以本地 SQLite 为源数据，`feishu_sync_jobs` 为可靠重试队列。本轮不破坏旧表结构，只扩展同步 payload 和自动建字段映射：

- 【智能参数】子表继续同步四个参数：绿化程度、城市化程度、活力程度、光照温暖度，并新增原图路径。
- 【对话改造】子表新增“输入你的感受”，对应页面里的 `chat_extra` 输入内容，并新增原图路径。
- 【自由创作】子表新增原图路径、元素布局图片路径、画布 JSON 路径、元素摆放明细 JSON。
- 【灵感创想】子表新增原图路径、创作画布图片路径、画布 JSON 路径、创作画布明细 JSON。
- 飞书同步器会在目标表缺字段时自动创建字段；同步失败仍进入本地失败/重试队列。

### 5. 线上同步配置与重试补丁

- 线上 `.env` 已补充：
  - `FEISHU_BITABLE_CHAT_SUMMARY_TABLE_ID=tblZkSIqni3IjZ9J`
  - `FEISHU_BITABLE_SLIDER_SUMMARY_TABLE_ID=tbl0b7OU7sommUqK`
- 补充后已重启服务并执行同步重试，原先两条“智能参数表 ID 缺失”的失败记录已恢复同步。
- 新增飞书 upsert 容错：如果本地队列保存的 `remote_record_id` 在飞书端已不存在，程序会自动改为插入新记录，避免 `record not found` 长期卡死。
