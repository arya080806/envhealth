## 二、关于名为 `????` 的子表

这个表不是新的实验模式，也不是患者主动产生的数据表。

## 四、飞书是否适合自动记录图片

飞书可以记录图片，但不建议把飞书作为唯一图片原件仓库。

### 方案 A：飞书文本 URL 索引

做法：

- 程序把图片路径转换成可访问 URL，例如：
  - `https://envhealth.cn/api/image/xxx.png`
  - `https://envhealth.cn/api/download/xxx.png`
- 飞书字段存 URL，而不是服务器本地路径。

优点：

- 实现简单，稳定性高。
- 不重复上传大图片，飞书表格轻。
- 研究者可以从表格点击打开图片。

缺点：

- 图片仍存放在服务器，服务器迁移时必须一起迁移 `uploads/` 和 `outputs/`。
- 如果 URL 完全公开，需要考虑患者隐私和访问权限。

适合：近期快速稳定落地。

### 方案 B：飞书附件字段

做法：

- 程序调用飞书开放平台，把生成图、原图、画布图上传为附件。
- 飞书多维表格字段直接显示图片/附件。

优点：

- 飞书里查看最直观。
- 图片跟着飞书记录走，研究者不用找服务器目录。

缺点：

- 实现复杂，需要文件上传、附件 token、失败重试、容量和权限管理。
- 大量图片会让飞书 Base 变重，长期科研积累可能卡顿。
- 后续如果要批量下载原始图片，仍需要额外导出。

适合：需要飞书页面内直接预览图片，但要接受更高维护成本。

### 方案 C：服务器科研归档文件夹 + Excel 索引

做法：

按参与者和会话生成标准目录：

```text
research_exports/
  20260612-1/
    abb35fe9eb0d_drag_park/
      original.png
      generated.png
      canvas.png
      canvas.json
      metadata.json
  index.xlsx
```

Excel 每行记录：

- 参与者编号、登记姓名、会话 ID、模式、场景、时间
- 原图路径、修改后生成图路径、操作/笔画图路径
- 参数、元素数量、情绪标签、输入感受
- 本地相对路径和可点击链接

优点：

- 最适合科研长期积累。
- 图片原件、JSON、Excel 索引都在同一套归档里，便于备份、审计、统计和离线分析。
- 不依赖飞书是否卡顿或字段是否改变。

缺点：

- 需要新增导出/定时归档功能。
- 研究者查看实时数据时不如飞书方便。

适合：作为科研主档案，强烈推荐。

### 方案 D：每次会话生成 Word/PDF 个案报告

做法：

每个完成作品生成一份 `.docx` 或 `.pdf`：

- 第一页：参与者、模式、场景、时间、参数
- 图片：原图、修改后图、操作/笔画图
- 文本：输入感受、情绪标签、元素列表、AI prompt 摘要

优点：

- 最适合人工质控、伦理归档、论文材料回看。
- 图片和文字排版清楚。

缺点：

- 不适合直接做统计分析。
- 文件数量多，需要配合 Excel 总索引。

适合：作为 Excel/文件夹归档的补充。

## 五、推荐方案

推荐采用“双轨制”：

1. 飞书做轻量索引和同步状态。
2. 服务器生成科研归档文件夹，Excel 作为主索引。

具体建议：

- 飞书字段记录：参与者、模式、场景、关键参数、元素数量、文本、图片 URL、同步状态。
- 服务器归档记录：原图、生成图、画布图、JSON、metadata。
- Excel 汇总表作为后续统计和论文分析主入口。
- 可选生成 Word/PDF 个案报告，用于质控和人工复核。

这样飞书不会承担大图片仓库的压力，也不会因为字段或附件失败导致科研数据不完整。

## 六、关于这些技术字段是否要保留

### 会话ID

建议保留，但可以隐藏。

原因：

- 每次创作/生成的唯一标识。
- 飞书同步用它做 upsert，避免重复插入。
- 后续排查图片、日志、草稿箱记录都靠它定位。

不要删除或改名；可以在视图中隐藏。

### 本地用户ID

建议保留，但可以隐藏。

原因：

- 程序数据库内部用户编号。
- 当参与者编号、姓名有重复或改名时，可以用于追溯。

日常研究查看不需要展示，可以隐藏。

### 汇总键

建议保留，但可以隐藏。

原因：

- 汇总表不是一会话一行，而是一名用户/一个汇总对象一行。
- `汇总键` 是同步唯一键，删除后汇总表可能重复或无法更新。

不要删除；可以隐藏。

### 最近会话ID

建议保留，但可以隐藏。

原因：

- 用于知道该用户最近一次操作对应哪个 session。
- 排查“为什么次数变化”时很有用。

不影响日常查看，可以隐藏。

## 九、当前结论

当前飞书表格不是最终科研归档结构，而是“程序同步镜像 + 运营调试表”。它能记录关键数值和文本，但当前没有真正把图片作为附件存入飞书；图片原件仍在服务器。

最稳妥的科研积累方案是：

- 飞书保留轻量索引和状态。
- 服务器自动生成标准文件夹归档。
- Excel 做主索引。
- Word/PDF 做人工质控报告。

这样后续即使飞书字段变动、网页卡顿或附件上传失败，科研原始材料也不会丢。
